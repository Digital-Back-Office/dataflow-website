## Introduction

Dataflow removes typical setup friction so teams can move from idea to production faster. Instead of juggling local environments, connections, and deployment pipelines, use a single platform that provides pre-configured workspaces, shared environments, and one-click deployment.

## Core Workflow

- Launch a development-ready workspace with VS Code and Jupyter configured: [Studio overview](https://docs.dataflow.zone/workspace/studio/overview/)
- Define a managed environment once and reuse it across tools: [Managed dependencies](https://dataflow.zone/product/managed-dependencies)
- Create connections and secrets centrally: [Connections docs](https://docs.dataflow.zone/workspace/studio/connections/)
- Orchestrate with Airflow using the same environment: [Workflows docs](https://docs.dataflow.zone/workspace/studio/workflows/)
- Deploy apps or DAGs with a click: [Deploy to production](https://dataflow.zone/product/deploy-to-production)

## Key Benefits

- Zero setup time: start building within minutes using a development-ready workspace.
- Environment consistency: the same container runs in development and production.
- Faster collaboration: share notebooks, DAGs, and apps without environment drift.

## Quick Example

Work in Jupyter to prototype, move code to VS Code for production, and schedule via Airflow — all using the same managed environment. This reduces debugging and deployment overhead dramatically. See the quickstart: [Quickstart guide](https://docs.dataflow.zone/get-started/quickstart-guide/).
